## P1 remediation and re-audit

The normal `write_sql_snapshot` path formerly cleared the marker directly.
On Unix, unlink could succeed before parent sync failed. The operation then
reported failure while the recovery marker was absent.

Production now passes marker clearing into `finish_repair_locked`. That
boundary syncs the prior committed state and restores the marker when
clear durability fails. The injected regression test proves all four required
facts together:

- the command returns the injected error;
- the published `memory.sql` remains valid;
- the Git snapshot is committed and replay is `Unchanged`;
- `.pending-snapshot` exists again before the error returns.

Independent re-audit verdict: `PASS`; P1 `0`; P2 `0`.
